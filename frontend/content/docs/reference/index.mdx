Every tool is scoped to the bearer key's Quill user and connected X account. Quill deliberately exposes no direct-publish tool: drafts and Article reviews must be explicitly approved before scheduling.

# Account and profiles

## `get_quill_status`

**Read tool.** Return the authenticated Quill account, connected X account, and active draft/scheduled/article/research counts. Call this first.

_No input._
## `get_profiles`

**Read tool.** Read the authenticated person's private writing and reply profiles. Use the writing profile for posts/articles and the reply profile for X replies.

_No input._
## `update_profile`

**Write tool.** Replace one private Quill profile. Only call when the human explicitly provides or approves the new profile; never overwrite a profile based only on old posts.

```json title="Input schema"
{
  "type": "object",
  "properties": {
    "kind": {
      "type": "string",
      "enum": [
        "writing",
        "reply"
      ],
      "description": "The profile to change."
    },
    "profile": {
      "type": "string",
      "minLength": 40,
      "maxLength": 20000,
      "description": "The approved private profile text."
    }
  },
  "required": [
    "kind",
    "profile"
  ],
  "additionalProperties": false,
  "$schema": "http://json-schema.org/draft-07/schema#"
```
