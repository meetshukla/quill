This is the exact deployed Quill MCP surface, generated from the server rather than maintained by hand. Every tool is scoped to the bearer key&apos;s Quill account and its X API connection.

Use a workflow [Skill](/docs/skills/research-replies) for normal work. Use these pages only when you need an exact input schema or tool-level behavior.

## Reference sections

- [Account and profiles](/docs/reference/account-profiles) — status, writing/reply profiles, and owned-post voice sync.
- [Research and replies](/docs/reference/research-replies) — corpus pagination, capture, Manual scan rules, and copy-only replies.
- [Media assets](/docs/reference/media) — durable owned images/video.
- [Post drafts and scheduling](/docs/reference/posts-scheduling) — posts, replies, quotes, threads, queue, and cancellation.
- [Native X Articles](/docs/reference/articles) — Article drafts, X review URLs, and scheduling.
- [Automations](/docs/reference/automations) — CTA and evergreen repost rules.

## Safety boundary

Quill intentionally has no `publish_now` tool. An agent can create a private draft, but must receive approval for the exact text, media, date, time, and timezone before scheduling it.
