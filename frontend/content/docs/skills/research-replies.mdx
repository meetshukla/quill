title: Research and replies
description: Read the full private corpus and prepare source-specific replies for a human to post.
---

Use the **Quill Research and Replies** skill for competitor research, source-grounded campaign work, manual capture intake, and reply suggestions.

1. Call `get_research_index` before planning against the corpus.
2. Use `search_research` with source/type/date filters. Follow `nextCursor` until it is `null` whenever the full matching set matters.
3. Use `capture_research` only for sources already obtained through an explicit action. Preserve the original URL and available media metadata.
4. Use `list_research_rules` and `save_research_rule` to inspect or explicitly change Manual scan keyword rules.
5. Call `prepare_replies`, then `get_next_replies`, only when a natural reply can point to one exact detail from the source.

Replies are copy-only. The MCP never posts them. Quill does not silently capture X while you scroll; capture comes from Manual scan or an explicit per-post action.

Portable package: [`skills/quill-research-replies`](https://github.com/meetshukla/quill/tree/main/skills/quill-research-replies).
