Quill MCP is the deployed interface to your private X writing system. It connects an agent directly to your own profile, X account, research corpus, media library, drafts, queue, and automations.

There is no shared workspace and no shared content. A co-founder uses their own Quill login, connects their own X account, and uses their own key. Every MCP call is scoped to that person.

## Start here

1. [Connect Quill MCP](/docs/quickstart) using your personal key.
2. Choose the workflow skill you need below.
3. Use the [MCP reference](/docs/reference) only when you need a precise tool schema.

## Workflow skills

Each skill is a small, portable instruction package for one Quill workflow. It assumes the `quill` MCP is already connected; it does not create another local service or profile folder.

- [Profiles and voice](/docs/skills/profiles) — safely manage private profiles and an owned-post voice corpus.
- [Research and replies](/docs/skills/research-replies) — work through the complete corpus and prepare human replies.
- [Post drafts](/docs/skills/posts) — draft posts, replies, quotes, threads, and schedule approved drafts.
- [Media assets](/docs/skills/media) — store owned images/video and attach them to Quill drafts or Articles.
- [Native X Articles](/docs/skills/articles) — create, inspect on X, and schedule a reviewed Article.
- [Automations](/docs/skills/automations) — configure explicitly approved CTA and evergreen repost rules.

The versioned packages live in [`skills/`](https://github.com/meetshukla/quill/tree/main/skills). Install only the skills your Codex or Claude workflow needs.

## Review-first by design

Quill MCP has no `publish_now` tool. An agent may create a draft, but it must receive explicit approval for the exact text, media, date, time, and timezone before it calls a scheduling tool. The Quill worker then publishes at that time even when no agent session is running.
